SpringBoot跨域解决方案

Harrison Lee 2021-01-27 后端框架 SpringBoot

当我们在前后端分离的项目中，常常需要解决跨域问题。跨域是为了解决浏览器的同源限制，浏览器之所以提出同源策略，是为了避免跨站请求伪造（Cross-Site Request Forgery，简称CSRF）攻击。所谓CSRF，就是攻击者利用受害者的身份来执行一些需要受害者授权的操作。例如用户访问了合法网站A，并且用户主机还保留用户的会话信息，那么当用户访问非法网站B，网站B可以利用用户关于网站A的会话信息向网站A的后台发起请求。在同源策略的限制下，在网站B中执行的脚本不能向其他网站发起请求，确保了安全。
