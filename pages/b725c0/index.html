Java8新日期API

在Java 1.0中，Java只提供了java.util.Date类来处理日期和时间，但是该类只能以毫秒的精度表示时间。更糟糕的是其易用性，比如年份的起始为1900年，月份的起始从0开始。在Java 1.1中，Date类中很多方法被废弃了，取而代之的是java.util.Calendar类，但是该类仍然有类似的问题和设计缺陷。关于Date和Calendar类，到底该使用哪一个？而有些特性又只在一个类中存在，比如DateFormat方法就只在Date类里有，同时DateFormat也有线程安全的问题。

# java.time包结构

为了解决上述问题，Java 8引入了java.time包，包含了很多新的类，如LocalDate, LocalTime, LocalDateTime, Instant, Duration和Period等，除此之外，还有几个子包，分别是：

  • java.time.chrono：此包提供历法相关的接口与实现。Java中默认使用的历法是ISO 8086日历系统，也就是我们常说的公历。此外还提供了几种其他历法，泰国佛教历，民国历，日本历和伊斯兰历。每种历法API分为三部分：XXXEra，XXXChronology和XXXDate。
  • java.time.format：此包提供了日期格式化的方法。format包中定义了时区名称，日期解析和格式化的各种枚举。此包中的类都是final的，都提供了线程安全的访问。
  • java.time.temporal：此包中定义了整个日期时间框架的基础，各种时间单位，时间调节器等。Java 8中日期时间类都是实现了temporal包中的事件单位（Temporal），时间调节器（TemporalAdjuster）和各种属性的接口。
  • java.time.zone：此包定义了时区转换的规则和方法。

# 日期/时间类

LocalDate表示日期，LocalTime表示时间，LocalDateTime表示使日期和时间的合体。LocalDateTime中大部分关于日期的操作都与LocalDate类似，关于时间的操作都与LocalTime类似。

# 时间点和时间间隔

时间点是时间线上的一个点，表示一个时间戳。时间间隔表示两个时间点之间的间隔。Instant表示时间点，Duration和Period表示时间间隔。

# Instant

Instant表示一个时间点，或时间戳，可以精确到纳秒。

# 获取Instant实例

可以通过工厂方法now()来获取时间点对象：

Instant now = Instant.now();
1

所有对Instant实例的修改都不会修改源对象，而是返回一个新的对象。

可以对Instant实例执行plusXXX，minusXXX操作：

plusSeconds(); minusSeconds();
plusMillis(); minusMillis();
plusNanos(); minusNanos();

Duration表示时间间隔，可以从两个时间点对象获取到时间间隔。可以通过工厂方法between()来获取Duration实例：

# Period

Period和Duration类似，但Period包含年份，月份和日期，而Duration只包含秒，毫秒，纳秒。Period和Duraton的很多方法都很类似，这里就不再赘述了，更多细节请参考官方DOC。

# 操纵日期/时间对象

可以通过withXXX，plusXXX，minusXXX方法修改日期/时间对象的某些属性，也可以采用通用的with()，plus()，minus()方法来获取源对象经过修改后的对象。这些通用方法需要指定一个单位来取代XXX，对于plus()和minus()，该单位的类型是TemporalUnit，该接口有一个枚举实现：ChronoUnit，可以很方便地指定单位。对于with()，需要指定一个TemporalField类型的参数，可以使用接口的枚举实现ChronoField。

笔记

ChronoUnit实现TemporalUnit，而ChronoField实现TemporalField。XXXUnit表示时间单位，如年月日时分秒等；而XXXField表示某个具体的时间的属性，如MONTH_OF_YEAR。前者仅仅是单位而已，而可以通过后者表示的属性从时间中取出属性值。

上述操纵方式都是比较简单和直接的，对于复杂操作可以使用调节器（TemporalAdjuster），工具类TemporalAdjusters中定义了很多调节器。这个调节器类似比较器一样，是策略模式的体系。

with方法接受一个调节器，如获取下一个工作日的日期：

final LocalDate now = LocalDate.of(2021, 1,16); // Friday
final TemporalAdjuster adjuster = TemporalAdjusters.ofDateAdjuster((temporal) -> {
    int addDays = 1;
    if (now.getDayOfWeek() == DayOfWeek.FRIDAY) addDays = 3;
    else if (now.getDayOfWeek() == DayOfWeek.SATURDAY) addDays = 2;
    return temporal.plus(addDays, ChronoUnit.DAYS);
});
final LocalDate nextWorkDay = now.with(adjuster);
System.out.println(nextWorkDay); // 2020-01-18

# 格式化和解析

DateTimeFormatter类中以及预定了很多格式的格式化器：

  • BASIC_ISO_DATE：20210107
  • ISO_DATE_TIME：2021-01-17T15:33:33.5644861
  • ISO_DATE：2021-01-17
  • ...

也可以通过ofPattern()工厂方法创建一个格式化器：

final LocalDateTime now = LocalDateTime.now();
final DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy/MM/dd HH:mm:ss");
System.out.println(formatter.format(now)); // 2021/01/17 15:37:38

以及通过DataTimeFormatterBuilder来构建格式化器：

final LocalDateTime now = LocalDateTime.now();
final DateTimeFormatter formatter = new DateTimeFormatterBuilder()
    .appendText(ChronoField.YEAR)
    .appendLiteral("-")
    .appendText(ChronoField.MONTH_OF_YEAR)
    .appendLiteral("-")
    .appendText(ChronoField.DAY_OF_MONTH)
    .appendLiteral(' ')
    .appendText(ChronoField.HOUR_OF_DAY)
    .appendLiteral("/")
    .appendText(ChronoField.MINUTE_OF_HOUR)
    .appendLiteral("/")
    .appendText(ChronoField.SECOND_OF_MINUTE).toFormatter();
System.out.println(formatter.format(now)); // 2021-January-17 15/52/24

# 时区和历法

先来了解一下几个关于时区的相关概念。

  • GMT（Greenwich Mean Time）：英国格林威治标准时间。
  • UTC（Universal Time Coordinated）：世界协调时，在时刻上接近于GMT时间。

世界各国都以UTC时间为基准，本地时间=UTC + 时区差。

时区表示方式包括固定时间偏移量和地理区域：

  • 地理区域：格式为"{洲}/{城市}"，如"Asia/Shanghai"。
  • 固定时间偏移量：一般使用'+'或者'-'然后加上四位数字，如"+0800"表示东八区，"-0500"表示西五区，则UTC+0800表示东八区的时间。

计算机中的时间戳表示的是GMT时间1970年1月1日 00:00:00 000到具体时间的毫秒数，与时区没有干系，也就是说北京的电脑和纽约的电脑同一时刻获得的时间戳是一致的。只有把时间戳解释为时间才会考虑时区。

ZoneId表示时区的ID，其有两个子类，ZoneOffset和ZoneRegion分别表示时间偏移量格式和地理区域格式。除了上面所说的表示两种方式的基本格式外，Java还支持其他格式，具体请参考DOC。

ZonedDateTime表示含有时区信息的日期时间，可以利用该类来表示一个具有时区信息的日期时间对象。

final LocalDateTime shanghai = LocalDateTime.now(ZoneId.of("+0800"));
/* 
注意这里仅仅是将shanghai中的时间贴上"-0500"时区的信息，不是将"+0800"的时间转换为此时对应的"-0500"时间。
*/
final ZonedDateTime newyork = ZonedDateTime.of(shanghai, ZoneId.of("-0500"));
System.out.println(shanghai); // 2021-01-17T17:16:26.941673 没有时区信息
System.out.println(newyork); // 2021-01-17T17:16:26.941673-05:00 含有时区信息

/* 仍然只是更改了时区信息而已，时间信息没有变化 */
ZonedDateTime zonedShanghai = shanghai.atZone(ZonedId.of("-0500")); // 2021-01-17T17:16:26.941673-05:00

newyork = ZonedDateTime.now(ZoneId.of("-0500"));
// 获取当前时刻其他时区的时间信息
System.out.println(newyork); // 2021-01-17T04:38:37.863443600-05:00

如果要将A时区的某个时间转化为对应那个时刻的B时区的时间，即时间的时区转换，需要使用ZonedDateTime的withZoneSameInstant()方法。

/* 整个转换过程可以简化为 ldtA->zdtA->zdtB->ldtB */
final LocalDateTime shanghai = LocalDateTime.now(ZoneId.of("+0800"));
final ZonedDateTime zonedShanghai = ZonedDateTime.of(shanghai, ZoneId.of("+0800"));
final ZonedDateTime zonedNewYork = zonedShanghai.withZoneSameInstant(ZoneId.of("-0500"));
final LocalDateTime newYork = zonedNewYork.toLocalDateTime();
System.out.println(shanghai); // 2021-01-17T17:48:27.590701600
System.out.println(newYork); // 2021-01-17T04:48:27.590701600
