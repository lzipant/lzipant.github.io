Java中的枚举

枚举类型是Java1.5新增的特性之一，但是可能平时很少使用，所以对枚举不是很熟悉。但是枚举在某些场景中很适用，所以本文就来分析一下Java中的枚举。

# 枚举的简单使用

public enum Mode{
  DAY_MODE, DARK_MODE
}
1
2
3

这样就定义好了一个枚举，其中enum关键字，这里Mode只是枚举的名称，和类名差不多。然后{}内部是枚举的值，一般用大写字母表示，多个值之间用逗号分割。

使用枚举也很简单，下面是一个简单的枚举使用案例：

public class ENUMTest {
  private final static int DAY_BEGIN_HOUR = 7;
  private final static int DAY_END_HOUR = 19;
  public static void main(String[] args) {
    Mode mode = (LocalTime.now().getHour() >= DAY_BEGIN_HOUR && LocalTime.now().getHour() < DAY_END_HOUR) ? Mode.DAY_MODE : Mode.DARK_MODE;
    System.out.println(getModeName(mode));
  }

  public static String getModeName(Mode mode) {
    if (mode == Mode.DAY_MODE) {
      return "白天模式";
    } else {
      return "夜间模式";
    }
  }
}

// 另一个文件中

  DAY_MODE, DARK_MODE

# 枚举的实现原理

上面的代码编译后，会生成一个字节码文件：Mode.class，感觉和类太像了。我们使用jad将其反编译一下，内容如下所示：

public final class Mode extends Enum
{

    public static Mode[] values()
    {
        return (Mode[])$VALUES.clone();
    }

    public static Mode valueOf(String name)
    {
        return (Mode)Enum.valueOf(com/color/se/enumeration/Mode, name);
    }

    private Mode(String s, int i)
    {
        super(s, i);
    }

    public static final Mode DAY_MODE;
    public static final Mode DARK_MODE;
    private static final Mode $VALUES[];

    static 
    {
        DAY_MODE = new Mode("DAY_MODE", 0);
        DARK_MODE = new Mode("DARK_MODE", 1);
        $VALUES = (new Mode[] {
            DAY_MODE, DARK_MODE
        });
    }
}

可以发现，和普通的类定义没有什么区别。其实Enum是所有枚举类型的公共基类，这是一个java.lang包下面的抽象类，而且具有泛型形参，不过上述反编译结果中并没有体现这一点。但是请注意，我们不能直接继承Enum类，否则无法编译。

另外，编译器还生成了values()和valueOf()方法以及构造器，但该构造器是私有的。

# 枚举的进阶用法

# 自定义构造函数和添加方法

我们修改一下上面简单使用的那个例子，当直接输出枚举常量就可以输出对应的中文含义：

public enum Mode {
  // 在枚举常量声明前不能声明方法
  DAY_MODE("白天模式"), DARK_MODE("夜晚模式"); // 注意分号

  private final String description;

  private Mode(String description) {
    this.description = description;
  }

  public String toString() {
    return this.description;
  }
}

# 枚举与switch

switch表达式支持枚举类型，下面来看一个使用例子：

Mode mode = (LocalTime.now().getHour() >= DAY_BEGIN_HOUR && LocalTime.now().getHour() < DAY_END_HOUR) ? Mode.DAY_MODE : Mode.DARK_MODE;
switch (mode) {
  case DAY_MODE:
    System.out.println("白天模式");break;
  case DARK_MODE:
    System.out.println("夜间模式");break;
  default:
    System.out.println("The mode is invalid.");
}

注意，在case后面没有指明枚举名，这是允许的。JVM会根据switch语句中传入的类型来自动推断。

笔记

switch除了支持枚举类型以外，还支持char，byte，short，int型，从1.7开始也支持String类型。case后面跟的是事实最终常量（effective constant），如果参数一旦与某个case后面的常量匹配，那么将会一直执行直到break的出现；当没有任何case匹配时，可以定义一组默认操作。

在使用switch ... case...语句时，特别注意是否添加break，否则会造成意想不到的效果。可以加上编译选项-Xlint:fallthrough来检查是否每个case分支后面都加了break，如果没有会抛出警告。

# 枚举实现接口和定义抽象方法

由于枚举生成的类中已经继承了Enum类，所以不能再继承了（Java的单继承限制），但是可以实现接口。笔者确实未能想出这种用法的案例，所以就不举例了，但是枚举确实能够实现接口的。

另外，枚举中也可以定义抽象方法，不过每个枚举常量必须实现这样该方法。

public enum Mode {
  DAY_MODE("白天模式") {
    @Override
    int getStartHour() {
      return 7;
    }
    @Override
    int getEndHour() {
      return 19;
    }
  }, 
  DARK_MODE("夜晚模式"){
    @Override
    int getStartHour() {
      return 19;
    }
    @Override
    int getEndHour() {
      return 7;
    }
  };

  private final String description;
  private Mode(String description) {
    this.description = description;
  }
  
  abstract int getStartHour();
  abstract int getEndHour();
  
  public String toString() {
    return this.description;
  }
}
