二叉查找树

二叉查找树，或二叉搜索树（Binary Search Tree, 简称BST），是一种特殊的二叉树结构，对其任意节点，其左子树中的任一节点的值都小于或等于当前节点的值，其右子树中的任一节点的值都大于或等于当前节点值（对于等于的情况，一般选择保存到任一边）。二叉查找树的中序遍历，是满足升序排列的。

下面探讨二叉搜索树的一些操作：

  • 构建BST
  • 插入节点
  • 查找节点
  • 删除节点

代码中使用到的BinTreeNode的定义请参考这篇博客。

# 插入节点

在探讨构建之前，先分析一下如何插入节点。插入节点，首先要找到所要插入的位置，通过上面的简述，发现无论如何，插入的节点最终都会成为一个叶子节点。那么可以通过遍历，分别比较当前节点值与待插入节点值，直到叶子节点位置。

具体的代码实现，其实可以用迭代来代替消除递归，设置一个节点指针node指向遍历过程中的节点，初始值为根节点。如果node节点的值大于待插入值，则将node指向node的左子节点，同理，如果大于或等于（这里把等于的情况放在右子树中，放在左子树中也可以），则将node指向node的右子节点。但是注意，在指针指向转移的时候，如果已经遍历到根节点了，则直接创建一个节点挂在当前的节点上。

/**
 * @param root 树的根节点
 * @param value 待插入值
 */
public void insert(BinTreeNode<T> root, T value) {
    assert Objects.nonNull(root);
    BinTreeNode<T> node = root;
    BinTreeNode<T> newNode = new BinTreeNode<>(value);
    while (true) {
        if (value.compareTo(node.getData()) < 0) {
            if (Objects.isNull(node.getLeft())) {
                node.setLeft(newNode);
                break;
            } else {
                node = node.getLeft();
            }
        } else {
            if (Objects.isNull(node.getRight())) {
                node.setRight(newNode);
                break;
            } else {
                node = node.getRight();
            }
        }
    }
}

# 构建BST

既然实现了插入节点，那么构建一棵BST就不是什么难事了，而且思路很直观，遍历值列表并调用插入节点方法即可。下面的实现其实比较粗糙，可能会导致BST链式化，最好取中位数的元素作为根节点。

/**
 * @param list 构建BST的值列表
 * @return 返回BST的根节点
 */
public BinTreeNode<T> build(List<T> list) {
    if (Objects.nonNull(list) && list.size() > 0) {
        BinTreeNode<T> root = new BinTreeNode<>(list.get(0));
        for (int i = 1; i < list.size(); i++) {
            insert(root, list.get(i));
        }
        return root;
    } else {
        return null;
    }
}

# 查找节点

根据插入节点的思路，查找节点的思路也很简单，除了插入这一步操作之外，其他都一样。

/**
 * @param root BST的根节点
 * @param target 所需要查找的值
 * @return 返回查找结果，如果查找成功，返回结果，否则返回null
 */
public T search(BinTreeNode<T>root, T target) {
    BinTreeNode<T>node = root;
    while (Objects.nonNull(node)) {
        int compare = node.getData().compareTo(target);
        if (compare == 0) {
            return node.getData();
        } else if (compare < 0) {
            node = node.getRight();
        } else {
            node = node.getLeft();
        }
    }
    return null;
}

# 删除节点

删除节点比较复杂一些，不过首先仍然需要找到待删除的节点，总体来说，可以分为三种情况：

  1. 待删除节点为叶子节点（左右子树都为空）
  2. 待删除节点左子树为空或右子树为空（即左右子树不同时为空）
  3. 待删除节点左右子树都不为空

对于代码实现，需要一个指针（node）指向待删除节点，还需一个指针（parent）指向待删除节点的父节点，以及一个布尔值变量（isLeftChild）表示待删除节点是其父节点的左子节点还是右子节点。node初始化为根节点，parent初始化为null。

  1. 对于第一种情况，如果是根节点，直接返回null；否则，根据isLeftChild从parent上摘掉待删除节点。
  2. 对于第二种情况，如果是根节点，返回非空的那个子节点；否则，根据isLeftChild将node的非空子节点挂在parent相应的位置上。
  3. 对于第三种情况，这里不用区分是根节点与否。有两种方式，第一，将被删除节点的左子树的最右边节点替换当前节点并删除，或者是将被删除节点的右子树的最左边节点替换当前节点并删除。这里采用第一种方式，但在查找左子树最右边节点的过程中需要注意判断替换节点是否就是待删除节点的左子节点。
/**
 * @param root BST的根节点
 * @param target 待删除节点
 * @return 返回删除后的BST的根节点
 */
public BinTreeNode<T> delete(BinTreeNode<T> root, T target) {
    BinTreeNode<T> node = root;
    BinTreeNode<T> parent = null;
    boolean isLeftChild = false;
    while (Objects.nonNull(node)) {
        int compare = node.getData().compareTo(target);
        // 找到目标节点并准备删除
        if (compare == 0) {
            // 当前节点为叶子节点
            if (Objects.isNull(node.getLeft()) && Objects.isNull(node.getRight())) {
                // 已排除当前节点为根结点的情况
                if (Objects.isNull(parent)) { // 当前二叉树只有一个节点，即需要删除根节点
                    return null;
                } else if (isLeftChild){
                    parent.setLeft(null);
                } else {
                    parent.setRight(null);
                }
            } else if (Objects.isNull(node.getLeft()) || Objects.isNull(node.getRight())) { // 左子节点为空或右子节点为空
                if (Objects.isNull(parent)) {
                    return Objects.isNull(node.getLeft()) ? node.getRight() : node.getLeft();
                }else if (isLeftChild) {
                    parent.setLeft(Objects.nonNull(node.getRight()) ? node.getRight() : node.getLeft());
                } else {
                    parent.setRight(Objects.nonNull(node.getLeft()) ? node.getLeft() : node.getRight());
                }
            } else { // 当前节点左右子节点都非空
                /*
                    有两种方法，即让当前节点的左子树的最右边节点取代当前节点
                    或者让当前节点的右子树的最左边节点取代当前节点
                     */
                parent = null; // parent记录的外部值以及无效，所以在这里可以复用
                BinTreeNode<T> tmp = node.getLeft();
                while (Objects.nonNull(tmp.getRight())) {
                    parent = tmp;
                    tmp = tmp.getRight();
                }
                // 这里需要判断所要删除节点的左子节点的右子节点是否存在，不同的情况处理方式不同
                if (Objects.isNull(parent)) { // 被删除节点的左子节点没有右子节点
                    node.setLeft(tmp.getLeft());
                } else {
                    parent.setRight(tmp.getLeft());
                }
                node.setData(tmp.getData());
            }
            break;
        } else if (compare < 0) {
            parent = node;
            node = node.getRight();
            isLeftChild = false;
        } else {
            parent = node;
            node = node.getLeft();
            isLeftChild = true;
        }
    }
    return root;
}

# 总结

虽然BST在一定程度上提高了查找的效率，但是在一些特殊的情况下，比如当BST成为链式结构时，查找时间复杂度线性化。

本文并没有分析如何查找多个相同的值，不过可以在上面的代码中做一些改进。例如，加入将相同的元素放在右子树中，那么当查找到一个相同的元素后，并不直接返回，而是保存到结果集中，然后继续沿着右子树的查找，直到叶子节点。
