AVL树

AVL树是一种平衡二叉树，也可以说是最“严格”的平衡二叉树，因为其要求左右子树高度差不超过1，相比其他的平衡二叉树，该条件更为严苛。平衡二叉树可以解决BST链式化的问题。

# 旋转

AVL树中最重要的操作是旋转，即插入和删除节点后，为了让树重新保持平衡，可能需要进行旋转调整。可以总结为四种旋转：LL, LR, RL, RR型，其中L表示左子树的高度比右子树的高度高，R表示右子树的高度比左子树的高度高。所以，在旋转之前，首先需要不平衡子树属于哪种情况。

# LL

如上图所示，在调整前以a点为根节点的子树不平衡，原因是其左子树的高度为h+2，而其右子树X高度仅为h。根节点左子树的高度比右子树高度高，且左子树根节点的左子树仍然比其右子树高度高，这种状态称为LL。

调整过程只需要让a的左子节点指向b的右子节点，且b的右子节点指向a，并让b点成为子树的根节点，类似这样的旋转称为右旋。

对应的实现代码如下所示（关于AVLTreeNode的定义以及相关方法请参考~其他相关代码~）：

private <T> void R(AVLTreeNode<T> root, AVLTreeNode<T> parent, Boolean isLeftChild) {
    AVLTreeNode<T> left = (AVLTreeNode<T>) root.getLeft();
    root.setLeft(left.getRight());
    left.setRight(root);
    if (isLeftChild) {
        parent.setLeft(left);
    } else {
        parent.setRight(left);
    }
    updateHeight(root); // root已经成为left的右子节点，必须先更新该节点的高度
    updateHeight(left);
}

# RR

如上图所示，在调整前，a节点的右子树与左子树的高度差为2，而且b节点仍然满足右子树比左子树高（高度差为1），这样的状态称为RR（RR与LL相互对称）。

调整的过程称为左旋，即将a节点的右子节点指向b节点的左子树，然后将b的左子节点指向a。

对应的代码实现如下所示：

private <T> void L(AVLTreeNode<T> root, AVLTreeNode<T> parent, Boolean isLeftChild) {
    AVLTreeNode<T> right = (AVLTreeNode<T>) root.getRight();
    root.setRight(right.getLeft());
    right.setLeft(root);
    if (isLeftChild) {
        parent.setLeft(right);
    } else {
        parent.setRight(right);
    }
    updateHeight(root); // root已经成为left的左子节点，必须先更新该节点的高度
    updateHeight(right);
}

# LR和LR

在理解了LL和LR的情况下，理解LR和RL就比较容易了。这里以LR类型为例，RL类型可以类比。

对于LR，需要先对根节点的左子树进行左旋操作，再对当前树做右旋操作。

根据根节点的左子树的右子树的拆分情况，又可以分为A, B, C三类。

# A

这种情况，拆分后的子树左高右低。在以b节点为根节点的子树做了左旋操作过后，得到以c点为根节点的子树，不过此时树仍然不平衡，之后对以a点为根节点的子树进行右旋操作，将c点作为根节点，a点作为c的右子节点，此时树才变为平衡，具体各子树的高度如上图所示。

# B

这种情况，拆分后的子树一样高。在以b节点为根节点的子树做了左旋操作过后，得到以c点为根节点的子树，此时该子树处于平衡状态，不过以点a为根节点的子树仍然不平衡，所以继续右旋，最终处于平衡状态，具体子树的高度如上图所示。

# C

这种情况，拆分后的子树左低右高。在以b节点为根节点的子树做了左旋操作过后，得到以c点为根节点的子树，此时该子树处于平衡状态，而且不同于前面两种情况，以点a为根节点的子树也已经处于平衡状态，当然，为了实现代码的统一性，继续进行右旋，旋转后树仍然是平衡的。

AVL树的删除和插入节点操作都会进行树的操作，所以在代码中把这一部分逻辑拆离出来，然后在插入或删除后统一地执行该操作。由于插入和删除都要先找到插入点或删除点，在查找的过程中会有一条path，而且发生不平衡的情况只会在该path上，所以在操作插入和删除时，保存该path记录，然后调用进行调整的方法。

下面是进行调整的方法的代码：

private <T> AVLTreeNode<T> adjust(Deque<AVLTreeNode<T>> path, Deque<Boolean> lefts) {
    // chroot为假根，将真实的根节点作为chroot的左孩子
    AVLTreeNode<T> parent, chroot = new AVLTreeNode<>();
    chroot.setLeft(path.peekFirst());
    Boolean isLeftChild;
    while (path.size() > 0) {
        parent = path.pollLast();
        isLeftChild = lefts.pollLast();
        updateHeight(parent);
        if (getBalanceFactor(parent) == 2) { // LL or LR
            if (getBalanceFactor((AVLTreeNode<T>) parent.getLeft()) == -1) { // LR
                L((AVLTreeNode<T>) parent.getLeft(), parent, true);
            }
            // LL型和LR型都需要下面的步骤进行调整
            R(parent, Objects.nonNull(isLeftChild) ? path.peekLast() : chroot,
              Objects.nonNull(isLeftChild) ? isLeftChild : true);
        } else if (getBalanceFactor(parent) == -2) { // RR or RL
            if (getBalanceFactor((AVLTreeNode<T>) parent.getRight()) == 1) { // RL
                R((AVLTreeNode<T>) parent.getRight(), parent, false);
            }
            // RR型和RL型都需要下面的步骤进行调整
            L(parent, Objects.nonNull(isLeftChild) ? path.peekLast() : chroot,
              Objects.nonNull(isLeftChild) ? isLeftChild : true);
        }
    }
    return (AVLTreeNode<T>) chroot.getLeft();
}

参数path指的是路径，lefts保存了path中每一个节点（除了第一个外）是其父节点的左孩子（true）还是右孩子（false），在写这篇博客的时候我发现没必要保存lefts，只需在操作节点时动态判断就可以了。

另外，有可能调整过后会改变根节点，然而真实的根节点没有根节点，所以定义了一个变量为假根，其左孩子始终指向真实的根节点。最后返回调整后的真实根节点。

上述代码并没有做优化，这方面我目前还没有分析，~好像只要在中途进行了调整过后，就不用再继续回溯到根节点了~。

对于插入操作和删除操作，与二叉搜索树类似。

# 插入节点

大体逻辑和BST的插入类似，不过需要将插入节点的父节点到根节点的路径保存下来。

public <T> AVLTreeNode<T> insert(AVLTreeNode<T> root, T value) {
    assert Objects.nonNull(value);
    Deque<AVLTreeNode<T>> path = new ArrayDeque<>();
    Deque<Boolean> lefts = new ArrayDeque<>();
    AVLTreeNode<T> node = root;
    AVLTreeNode<T> newNode = new AVLTreeNode<>(value, 1);
    while (Objects.nonNull(node)) {
        path.addLast(node);
        if (value.compareTo(node.getData()) < 0) {
            if (Objects.isNull(node.getLeft())) {
                node.setLeft(newNode);
                break;
            } else {
                node = (AVLTreeNode<T>) node.getLeft();
                lefts.addLast(true);
            }
        } else {
            if (Objects.isNull(node.getRight())) {
                node.setRight(newNode);
                break;
            } else {
                node = (AVLTreeNode<T>) node.getRight();
                lefts.addLast(false);
            }
        }
    }
    if (path.size() == 0) {
        return newNode;
    } else {
        return adjust(path, lefts);
    }
}

# 删除节点

注意，删除节点时，在第三种情况（左右子树都不为空），要将其节点本身也保存到path栈中。

public AVLTreeNode<T> remove(AVLTreeNode<T> root, T value) {
    Deque<AVLTreeNode<T>> path = new ArrayDeque<>();
    Deque<Boolean> lefts = new ArrayDeque<>();
    AVLTreeNode<T> node = root;
    int compare;
    while (Objects.nonNull(node)) {
        compare = value.compareTo(node.getData());
        if (compare == 0) {
            if (Objects.isNull(node.getLeft()) && Objects.isNull(node.getRight())) {
                if (node == root) {
                    return null;
                } else if (lefts.peekLast()) {
                    path.peekLast().setLeft(null);
                } else {
                    path.peekLast().setRight(null);
                }
            } else if (Objects.isNull(node.getLeft()) || Objects.isNull(node.getRight())) {
                if (node == root) {
                    return (AVLTreeNode<T>) (Objects.nonNull(node.getLeft()) ? node.getLeft() : node.getRight());
                } else if (lefts.peekLast()) {
                    path.peekLast().setLeft(Objects.nonNull(node.getLeft()) ? node.getLeft() : node.getRight());
                } else {
                    path.peekLast().setRight(Objects.nonNull(node.getRight()) ? node.getRight() : node.getLeft());
                }
            } else {
                AVLTreeNode<T> parent = node;
                AVLTreeNode<T> tmp = (AVLTreeNode<T>) node.getLeft();
                while (Objects.nonNull(tmp.getRight())) {
                    path.addLast(parent);
                    parent = tmp;
                    tmp = (AVLTreeNode<T>) tmp.getRight();
                }
                if (tmp == parent.getLeft()) {
                    parent.setData(tmp.getData());
                    parent.setLeft(tmp.getLeft());
                    path.addLast(parent); // 注意这一步
                } else {
                    node.setData(tmp.getData());
                    path.peekLast().setRight(tmp.getLeft());
                }
            }
            node = null;
        } else if (compare < 0) {
            path.addLast(node);
            lefts.addLast(true);
            node = (AVLTreeNode<T>) node.getLeft();
        } else {
            path.addLast(node);
            lefts.addLast(false);
            node = (AVLTreeNode<T>) node.getRight();
        }
    }
    return adjust(path, lefts);
}

# 其他相关代码

@Getter
@Setter
@ToString
@NoArgsConstructor
static class AVLTreeNode<T> extends BinTreeNode<T> {
    int height; // 以当前节点为根节点的子树的高度

    public AVLTreeNode(T value, int height) {
        super(value);
        this.height = height;
    }
}

private int getHeight(AVLTreeNode<T> node) {
    if (Objects.isNull(node)) {
        return 0;
    } else {
        return node.getHeight();
    }
}

private void updateHeight(AVLTreeNode<T> node) {
    assert Objects.nonNull(node);
    node.setHeight(Math.max(getHeight((AVLTreeNode<T>) node.getLeft()),
                            getHeight((AVLTreeNode<T>) node.getRight())) + 1);
}

private int getBalanceFactor(AVLTreeNode<T> node) {
    assert Objects.nonNull(node);
    return getHeight((AVLTreeNode<T>) node.getLeft()) - getHeight((AVLTreeNode<T>) node.getRight());
}
