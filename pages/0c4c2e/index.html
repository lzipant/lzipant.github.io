Spring AOP基础

AOP是Spring的两大核心之一，在Spring种占据重要地位。但是由于其涉及很多复杂概念，配置方式多种多样，常常让学习者眼花缭乱，晕头转向。本文对AOP相关的概念和配置做一个简单的梳理。

# 应用场景

  • 记录日志
  • 监控方法的运行（如运行时间，入参，返回值等）
  • 权限控制
  • 缓存优化
  • 事务管理

# 相关概念

  • 连接点（Join Point）：指程序执行的某个特定位置，Spring仅支持方法的连接点。即仅能在方法调用前，方法调用后，异常抛出时及方法调用前后这些程序执行点织入增强。连接点是程序类中客观存在的事务。

  • 切点（Pointcut）：指增强要织入的具体位置，即连接点的信息，切点是连接点，而连接点不一定是切点。

  • 增强（Advice）：又被称为通知。既包含了一段在切点上执行的逻辑，又包含了在众多连接点中定位切点点的方位信息。

  • 目标对象（Target）：指增强逻辑的织入目标。如对象A中有一个方法sayHello()，如果该方法是一个切点，那么对象A就是AOP的目标对象。

  • 织入（Weaving）：将目标类，增强编织在一起。Spring AOP有3种织入方式：

    • 编译期织入，要求使用特殊的编译器。
    • 类装载期织入，要求使用特殊的类装载器。
    • 动态代理织入，在运行期为目标类添加增强生成子类的方式。

    Spring AOP采用动态代理织入，而AspectJ采用编译期织入和类装载期织入。

  • 代理（Proxy）：进行织入后，目标对象会生成一个结果类。

  • 切面（Aspect）：可以仅通过增强类生成一个切面，但切点必须结合增强才能制作出切面。增强包含连接点信息和横切逻辑，但切点仅包含连接点信息（感觉有点不好理解？？？）。

# 实现者

AOP的设计目标是把横切的问题模块化。在Java中，相关的AOP实现有AspectJ，Spring AOP等。前者是Java领域AOP的完全解决方案，十分强大；而后者是Spring提供的轻量级方案，致力于解决企业级开发中最普遍的需求。

# 使用配置

目前Spring AOP一共有三种配置方式：

  • 基于接口的配置（始于Spring 1.2）。
  • 基于schema的方式（始于Spring 2.0）。
  • @Aspect注解配置（始于Spring 2.0）。

为了解决上面说到的问题，使用BeanNameAutoProxyCreator代替ProxyFactoryBean。

  <property name="interceptorNames">
    <list>
      <value>logArgsAdvice</value>
      <value>logResultAdvice</value>
    </list>
  </property>
  <property name="beanNames" value="*ServiceImpl"/>
</bean>

# 基于schema的配置

在上面基于@Aspect配置的基础上，去掉Java类中的相关注解（因为这种配置方式不需要用到任何注解），然后在xml中配置如下：

<bean id="userServiceImpl" class="net.harrison.service.impl.UserServiceImpl" />
<bean id="logArgsAdvice" class="net.harrison.advice.LogArgsAdvice"/>
<bean id="logResultAdvice" class="net.harrison.advice.LogResultAdvice"/>

<aop:config>
    <!-- 第一种方式 -->
    <aop:pointcut id="logArgsPointCut" expression="execution(* net.harrison.service..*.*(..))"/>
    <aop:aspect ref="logArgsAdvice">
        <aop:before pointcut-ref="logArgsPointCut" method="logArgs"/>
    </aop:aspect>
    
    <!-- 第二种方式 -->
    <aop:aspect ref="logResultAdvice">
        <aop:pointcut id="logResultPointCut" expression="execution(* net.harrison.service..*.*(..))"/>
        <aop:after-returning method="afterReturning" pointcut-ref="logResultPointCut" returning="result"/>
    </aop:aspect>
</aop:config>

# 创建增强

上面配置中，只是简单的使用了两种增强类型，在Spring AOP中还支持其他类型的增强。下面详细分析一下各种类型的增强的使用方式。

下图是增强接口继承关系图：

其中<<aopalliance>>是指aop联盟定义的，<<spring>>是指Spring定义的。

在Spring AOP中，一共有5中增强类型：

  • 前置增强，MethodBeforeAdvice接口。
  • 后置增强，AfterReturningAdvice接口。
  • 环绕增强，MethodInterceptor接口。
  • 异常抛出增强，ThrowAdvice接口。
  • 引介增强，IntroductionInterceptor接口。

可以实现实现这些接口来定义增强。

但是使用@Aspect注解配置时，一般使用注解来定义增强，一般包括三部分：增强类型，目标切点表达式和增强横切逻辑。增强横切逻辑一般就是方法，增强类型是注解，目标切点表达式为这些注解的参数，是一些切点表达式函数。

  • 前置增强：@Before。
  • 返回增强：@AfterReturning。方法正常返回时执行。
  • 后置增强：@After，不管是抛出异常还是正常退出，该增强都会得到执行。
  • 异常抛出增强：AfterThrowing。方法抛出异常时执行。
  • 环绕增强：@Around。
  • 引介增强：@DeclareParents。

切点表达式由切点表达式函数组成，Spring支持9中不同的切点表达式函数，用不同的方式描述切点信息，根据描述对象的不同，可以大致分为4类：

  • 方法切入点函数：通过描述目标类方法的信息定义连接点。
  • 方法入参切点函数：通过描述目标类方法入参的信息定义连接点。
  • 目标类切点函数：通过描述目标类类型的信息定义连接点。
  • 代理类切点函数：通过描述目标类的代理类的信息定义连接点。

# 切点类型

Spring定义了6种类型的切点：

  • 静态方法切点
  • 动态方法切点
  • 注解切点
  • 表达式切点
  • 流程切点
  • 复合切点

# 切面类型

  • Advisor：表示一般切面。
  • PointcutAdvisor：具有切点的切面。可以通过PointcutAdvisor的实现类来创建切面。
  • IntroductionAdvisor：引介切面。
