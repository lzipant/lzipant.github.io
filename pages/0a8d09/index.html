Java中的反射

反射是Java中很重要的机制，虽然业务代码中很少用反射，但是各种框架中几乎都用到了反射。我们虽然不怎么使用反射，但必须要知道反射是怎么实现的。

# 使用反射

我们写的Java类在经过编译后，会生成字节码文件存储在磁盘上。在运行时，当第一次用到该类时会加载类，类加载后在内存中存在一个对应的Class对象，该对象是类的运行时表示。

所以反射的第一步，就要获取Class对象。

# 获取Class对象

  • 通过数据类型的Class属性来获取：

    Class<Integer> clazz1 = int.class;
Class<Integer> clazz2 = Integer.class;
System.out.println(clazz1.getName()); // int
System.out.println(clazz2.getName()); // java.lang.Integer
Class<Void> clazz3 = void.class;
System.out.println(clazz3.getName()); // void

在获取了Class对象后，就可以反射创建实例，调用方法，获取字段等。

# 创建实例

  • 通过Class对象的newInstance()方法：

    Class<Integer> clazz1 = Integer.class;
Integer integer;
try {
    integer = clazz1.newInstance();
} catch (InstantiationException | IllegalAccessException e) {
    e.printStackTrace();
}

  • 通过Constructor来构造，首先需要获取Constructor实例。如果Class对象表示的是接口，基本类型，数组或void，那么下面两个方法返回的数组的长度为0。

    • getConstructors()：返回对应Class中的所有public构造器。
    • getDeclaredConstructors()：返回对应Class中所有的构造器，不管访问权限。

    另外还有一些返回单个Constructor的方法，需要指定构造器的参数列表。如果Class是定义在非静态上下文中的内部类，那么形参列表的第一个参数表示外围类的Class对象。

    • getConstructor()：返回指定构造器，只能返回满足条件的public的构造器。
    • getDeclaredConstructor()：返回指定构造器，不受访问权限的约束。

    另外还有一个相关方法：

    • getEnclosingConstructor()：如果Class是定义在某个构造器中的局部内部类或匿名内部类，那么返回这个外围构造器，如果不存在则返回null。

    获取到Constructor后，可以调用其newInstance()方法来创建实例。但是注意，如果该Constructor表示的是非静态上下文中内部类的构造器，那么第一个参数需要指定为外围类的实例对象。

# 获取字段

和构造器一样，获取字段也有两组方法，要么返回多个要么返回一个。先来看返回多个的情况：

  • getFields()：返回类或接口中public的所有字段（包括父类或父接口），如果没有public的，则返回空数组。

  • getDeclaredFields()：返回所有不包含继承的字段，但包含所有权限修饰符所修饰的字段。获取到的private字段可以直接访问。

对于上面两个方法，如果Class表示数组，基本类型或void，那么返回空数组。

另外还有获取单个字段的方法，含义和上面两个方法类似。

  • getField() ：根据名称，返回public字段（包含继承到的）。
  • getDeclaredField()：根据名称，返回不包含继承的字段（不受权限修饰符的影响）。

# 获取方法

Class类提供了返回一个或多个Method的方法。下面两个方法会返回Method数组：

  • getMethods()：如果Class对象表示普通类，返回所有public方法（包括继承到的）；如果Class对象表示数组，返回Object类中除了clone()方法的所有方法；如果Class对象表示接口，返回的Method数组中不包含Object类中的方法。如果接口中没有声明方法，且其继承链上的接口也没有声明方法，那么返回空数组；注意对于类总是会从Object类中继承方法，所以不会返回空数组。返回的方法中不包含"<init>"和"<cinit>"方法。
  • getDeclaredMethods()：和getMethods()类似，会返回直接声明的所有方法（即不受访问权限修饰符约束），但是不包含通过继承得到的方法。

下面三个方法会返回单个Method对象：

  • getMethod()：根据方法名和参数列表获取public方法，如果参数列表指定为null，会被看作空数组。继承规则和getMethods()一样。
  • getDeclaredMethod()：和getMethod()类似，但是无法获取继承而来的方法，且不受权限修饰符的约束。
  • getEnclosingMethod()：如果当前Class对象表示方法中的本地或匿名内部类，那么返回包含该内部类的方法；对于其他情况，返回null。

不管对于Constructor，Field还是Method，如果当前对这些对象各自所表示的目标不具有访问权限，那么可以通过调用各自的setAccessible(true)方法，可以绕过JVM的权限检查。

笔记

关于 setAccessible(true)是否破坏了private的安全性。首先，private不是用来保证字段或方法的安全性的，而是为了面向对象思想中的封装。private只是约束开发者遵守面向对象，而不是为了保证不能访问。

# Method

Method有一个方法叫invoke，这是Java反射的精髓，用来实现方法的被动调用。我们在SpringBoot项目中写的handlerMethod就是这样被调用的。

在JVM中有一个反射膨胀（inflation）机制：由于很多情况下反射只会调用一两次，所以JVM设置了一个名为sun.reflect.inflation.Threshold阈值，默认等于15。当反射方法的调用次数超过15次时，会使用ASM生成新的类，保证后面调用比native要快。当调用次数小于15时，直接用native的方式来调用，没有额外类的生成，校验，加载的开销。另外，还可以通过参数sun.reflect.noInflation来设置是否禁用反射膨胀机制，默认值为false。
