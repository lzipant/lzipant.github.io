Java中的AtomicInteger

AtomicInteger提供了很多原子性的操作方法，另外还有AtomicBoolean，AtomicLong，但原理都是类似的。本文以AtomicInteger为例，分析基本使用和底层原理。

# 基本使用

# 创建AtomicInteger

AtomicInteger提供了两个构造器，一个无参，默认初始值为0；另一个有一个参数，为所指定的初始值。

public AtomicInteger() {}
public AtomicInteger(int initialValue) {
    value = initialValue;
}

# Incremental操作

getAndIncrement()和incrementAndGet()

两者都会执行自增操作，但是前者返回增长前的值，而后者返回增长后的值。

public static void main(String[] args) {
    /*
    AtomicInteger虽然执行的是原子操作，但是如果atomicInteger被多个线程共享，
    那么下面两条输出语句的结果可能不如预期。
    */
    AtomicInteger atomicInteger = new AtomicInteger(); // 初始值为0
    System.out.println(atomicInteger.getAndIncrement()); // 0
    System.out.println(atomicInteger.incrementAndGet()); // 2
}

# decremental 操作

getAndDecrement()和decrementAndGet()

两者都会执行自减操作，但是前者返回减少前的值，而后者返回减少后的值。

public static void main(String[] args) {
    AtomicInteger atomicInteger = new AtomicInteger(10); // 初始值为10
    System.out.println(atomicInteger.getAndDecrement()); // 10
    System.out.println(atomicInteger.decrementAndGet()); // 8
}

# 更新值

  1. 可以调用下面方法来原子性地更新值，expectedValue表示当前值，newValue是需要设置的新值。该方法返回一个boolean类型的结果，当expectedValue和当前值不相等时，修改失败，返回false；否则返回true，表示修改成功。

    boolean compareAndSet(int expectedValue, int newValue)
    1

  2. 另外还有两个方法可以原子性地修改值：

int getAndAdd(int delta)
int addAndGet(int delta)

2

两者都会执行修改操作，修改后的值为当前值与delta之和，不过前者返回修改前的值，后者返回修改后的值。

  1. 另外还有两个支持lambda表达式的方法：

    int getAndUpdate(IntUnaryOperator updateFunction)
int updateAndGet(IntUnaryOperator updateFunction)

    public static void main(String[] args) {
    AtomicInteger atomicInteger = new AtomicInteger(10);
    System.out.println(atomicInteger.getAndUpdate(x -> x * x)); // 10
    System.out.println(atomicInteger.updateAndGet(x -> (int)Math.sqrt(x))); // 10

# 原理

几乎所有的原子类都调用了Unsafe类，该类被命名为不安全的，因为其功能很强大，如果使用不当会导致不可预料的后果。

# compareAndSwapInt()

该方法内部实现了经典的CAS算法，该方法位于Unsafe类中，是一个native方法。

/**
o: 原子类对象
offset: 原子类对象中value值的偏移量
expected: 期望的value的当前值
x: 新值
*/
public final native boolean compareAndSetInt(Object o, long offset,
                                                 int expected,
                                                 int x);

在HotSpot的实现源码中，有一段关键代码：

LOCK_IF_MP(%4) "cmpxchgl %1, (%3)"
1

大概意思是当在多处理器中会对cmpxchgl指令加上lock前缀。

警告

下面内容的真实性未经验证！！！

cmpxchgl指令为cmpxchg的变种，两者都差不多，另外还有一个指令叫做xchg。

xchg表示交换两个操作数，是原子指令。

cmpxchg表示比较并交换两个操作数，不是原子指令，所以在多核CPU中为了原子性要加lock前缀。

cmpxchg将al\ax\eax\rax中的值与第一操作数进行比较：

  • 如果相等，把第二操作数直接装载到第一操作数，zf置为1。
  • 如果不相等，第一操作数的值装载到al\ax\eax\rax，并将zf清0。

更多细节请参考这个pdf。

# getAndAddInt()

compareAndSwapInt()方法可能存在修改失败的情况，但是有时候必须要修改成功，比如调用incrementAndGet()，addAndGet()等方法，所以需要使用Unsafe类中getAndAddInt()方法。下面是该方法的源码：

public final int getAndAddInt(Object o, long offset, int delta) {
    int v;
    do {
        /* 通过字段偏移量的方式读取内存获取由volatile修饰的value值 */
        v = getIntVolatile(o, offset);
        /* 循环执行weakCompareAndSetInt方法，直到修改成功或超过虚拟机设置的自旋参数 */
    } while (!weakCompareAndSetInt(o, offset, v, v + delta));
    return v;
}
