从遍历结果中构建二叉树

在大部分教科书上都说明了前序+后序的遍历结果不能唯一地确定一棵二叉树，而前序+中序以及后序+中序则可以。本文则分析如何用代码来实现，关于为什么能够和不能够不做证明（主要是我太菜了 😂）。

# 前序+中序

这棵二叉树的前序，中序分别为：

  • 前序：0, 1, 3, 7, 4, 2, 5, 6
  • 中序：7, 3, 1, 4, 0, 5, 2, 6

前序遍历序列的第一个元素为根节点，然后在中序遍历中找到该元素，即可把中序遍历分成两部分，进而也把前序遍历分成两部分，然后递归进行处理。

根据上述的简要分析，可知函数的参数主要为前序遍历序列，中序遍历序列，以及各自的起始和结束下标。进入函数后首先根据下标判断序列是否存在元素，如果不存在，则代表是递归终点，否则则根据上述分析进行处理。

    /**
     * 参数校验应该在外部完成
     * @param pre 前序序列
     * @param a 前序序列开始下标
     * @param b 前序序列结束下标
     * @param in 中序序列
     * @param c 中序序列开始下标
     * @param d 中序序列结束下标
     * @return 生成树根节点
     */
public BinTreeNode<T> generateFromPreAndInOrder(List<T> pre, int a, int b, List<T> in, int c, int d) {
    if (a > b) {
        return null; // 递归终点
    } else {
        int index = c;
        while (index <= d && !Objects.equals(pre.get(a), in.get(index))) {
            ++index;
        }
        BinTreeNode<T> node = new BinTreeNode<>();
        node.setData(pre.get(a));
        // 递归处理左子树
        node.setLeft(generateFromPreAndInOrder(pre, a + 1, a + (index - c), in, c, index - 1));
        // 递归处理右子树
        node.setRight(generateFromPreAndInOrder(pre, a + (index - c) + 1, b, in, index + 1, d));
        return node;
    }
}

上面的函数由于是递归调用的，所有就没有对序列的合法性进行判断，外部调用必须确保前序遍历序列和后序遍历序列要相互对应，否则该函数不能正常运行，下面两类问题的实现也一样。

另外还有一点需要注意，在递归处理左子树时，前序遍历序列的结束下标应该为：a+1+[(index−c)−1]a + 1 + [(index - c) - 1]a+1+[(index−c)−1], 上述代码中为简化后的写法，前面这个式子更好理解一些（下面右子树的参数也可给予上述式子理解）。

# 后序+中序

对于后序+中序的情况也类似，只不过根节点不是第一个元素，而是最后一个元素。

/**
     * 参数校验应该在外部完成
     * @param post 后序序列
     * @param a 后序序列开始下标
     * @param b 后序序列结束下标
     * @param in 中序序列
     * @param c 中序序列开始下标
     * @param d 中序序列结束下标
     * @return 生成树根节点
     */
public BinTreeNode<T> generateFromPostAndInOrder(List<T> post, int a, int b, List<T> in, int c, int d) {
    if (a > b) {
        return null;
    } else {
        int index = d;
        while (index >= c && !Objects.equals(post.get(b), in.get(index))) {
            --index;
        }
        BinTreeNode<T> node = new BinTreeNode<>();
        node.setData(post.get(b));
        node.setLeft(generateFromPostAndInOrder(post, a, a + (index - c) - 1, in, c, index - 1));
        node.setRight(generateFromPostAndInOrder(post, a + (index - c), b - 1, in, index + 1, d));
        return node;
    }
}

# 层序+中序

仍然对于上图中的二叉树，其层序遍历和中序遍历分别是：

  • 0, 1, 2, 3, 4, 5, 6, 7
  • 7, 3, 1, 4, 0, 5, 2, 6

层序遍历序列的第一个元素，是当前二叉树的根节点，同样地，在中序遍历中找到该节点的下标，从而可以确定左右子树的中序遍历。如果能够继续求得左右子树的层序遍历就更好了，因为这样就可以利用递归来处理问题。由于左右子树的层序遍历在原始层序遍历中是分散开的，所以不能直接指定下标。但是可以从当前层序遍历中筛选，即正向遍历当前层序遍历，然后判断其是否在当前中序遍历范围内以及是属于左子树还是右子树。最后，类似前面两种情况，递归求解。

/**
     * 参数校验应该在外部完成
     * @param layer 层序序列
     * @param in 中序序列
     * @param a 中序序列开始下标
     * @param b 中序序列结束下标
     * @return 生成树根节点
     */
public BinTreeNode<T> generateFromLayerAndInOrder(List<T> layer, List<T> in, int a, int b) {
    if (a > b) {
        return null;
    }
    int index = a;
    while (index <= b && !Objects.equals(layer.get(0), in.get(index))) {
        ++index;
    }
    List<T> leftLayer = new ArrayList<>(index - a);
    List<T> rightLayer = new ArrayList<>(b - index);
    boolean isLeft;
    for (int i = 1; i < layer.size(); ++i) {
        isLeft = false;
        for(int j = a; j < index; ++j) {
            if (Objects.equals(layer.get(i), in.get(j))) {
                isLeft = true;
                break;
            }
        }
        if (isLeft) {
            leftLayer.add(layer.get(i));
        } else {
            rightLayer.add(layer.get(i));
        }
    }
    BinTreeNode<T> node = new BinTreeNode<>();
    node.setData(in.get(index));
    node.setLeft(generateFromLayerAndInOrder(leftLayer, in, a, index - 1));
    node.setRight(generateFromLayerAndInOrder(rightLayer, in, index + 1, b));
    return node;
}
