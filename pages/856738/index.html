JavaScript中的变量与作用域

本文来分析JavaScript中的变量。使用变量之前要进行声明，在JS中可以使用var，let和const（后两者是ES6引入的）。

# 变量提升

对于var，所声明的变量有两个作用域：全局作用域和函数作用域。对于全局作用域中声明的变量会绑定到window对象上，造成全局污染。

var a = 1
console.log(a === window.a) // true
1
2

另外var声明具有变量提升机制。可以在变量声明之前访问该变量。

console.log(a) // undefined
var a = 10
1
2

上面的代码被JS引擎解析后的执行步骤变成了下面这样子：

var a
console.log(a) // undefined
a = 10
1
2
3

上面这种效果就被称为变量提升（hoisting），另外在函数作用域中也存在变量提升：

function foo() {
  if(condition) {
    var a = 10
    console.log(a)
  } else {
    console.log(a)
  }
}

最好别使用这种方式和var关键字，使用let和const代替。

# 块级作用域

let和const在全局作用域中声明的变量，不会绑定到window对象上。

let b = 2;
const c = 3;
console.log(window.b === b); // false
console.log(window.c === c); // false

另外不管是在全局或函数作用域中，也不具有var的那种变量提升效果。

console.log(a) // ReferenceError
let a = 10

function foo() {
  if(condition) {
    var a = 10
    console.log(a)
  } else {
    console.log(a) // ReferenceError
  }
}

实际上，JavaScript引擎在扫描代码发现变量声明时，要么将它们提升至作用域顶部（遇到var声明），要么将声明放入暂时性死区（TDZ）中（遇到let和const声明时）。访问TDZ中的变量会触发运行错误，只有执行过变量声明语句后，变量才会从TDZ中移出，才可正常访问。

let和const具有块级作用域，通常指函数和由{}包裹的区域。

let msg = "msg"
function fun() {
    console.log(msg)
    let msg = "msg1"
}

注意，上述代码会报错，和其他很多语言不一样，JS中块级作用域内不能存在先使用后声明的变量。

所以，下面两种修改都是OK的。

let msg = "msg"
function fun() {
    console.log(msg) // output: msg
}

const和let类似，不过前者是用来定义常量的，这种常量直到被回收时都不能改变值。当该值类型是引用时，可以更改该引用中的值，但是不能指向其他引用。

可以发现，JavaScript中关于变量声明是非常复杂的。在使用时遵循先声明后使用的原则，尽量不要使用var。

# for循环中的块级作用域

var i = 100
for(var i = 1; i < 5; ++i) {
    console.log(i)
}
console.log(i) // output: 5
1
2
3
4
5

注意，和其他很多语言不一样的是，for循环里面的var变量声明没有块级作用域，导致修改了全局变量。只需要把for循环处的var改为let即可。

var i = 100
for(let i = 1; i < 5; ++i) {
    console.log(i)
}
console.log(i) // output: 100

再次说明，不要使用var。（太难了~ 😢）
